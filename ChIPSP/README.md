ChIPSP: Core Spatial Integration of ChIP-seq and Hi-C Data
Overview

ChIPSP is an R package that implements the core ChIP-SP algorithm for integrating ChIP-seq transcription factor binding with Hi-C chromatin interaction data to identify spatially linked regulatory regions.

The package focuses exclusively on the core computational engine:

Step 1: Merging Hi-C loop outputs across replicates and resolutions

Step 2: Spatial linking of ChIP-seq peaks to distal Hi-C loop anchors and ranking interactions

Downstream analyses (gene annotation, pathway enrichment, and visualization) are intentionally excluded from the package API and are instead provided as reference R scripts, allowing users full flexibility in annotation strategies.

Conceptual Framework

Hi-C identifies chromatin loops but does not resolve precise nucleotide-level contact points. ChIPSP therefore:

Identifies ChIP-seq peaks overlapping one anchor of a Hi-C loop

Assigns the partner anchor as a spatially linked regulatory region

Treats the entire interacting Hi-C bins as potential transcription factor effect sites

Spatial interactions are ranked based on:

ChIP-seq peak strength (pileup)

Hi-C loop confidence (FDR)

The resulting ranked regions can be directly used for:

ChIPpeakAnno-based gene annotation

UCSC Genome Browser visualization

Pathway and transcription factor enrichment analyses

From GitHub

devtools::install_github("Lattesnow/ChIP_SP")


After installation:

library(ChIPSP)

Input Requirements

All input files should be placed in the same working directory.

Hi-C Loop Files

Naming pattern:

*HiC.xls


May include:

Multiple replicates

Multiple loop-calling resolutions

ChIP-seq Peak File

Naming pattern:

*ChIP.xls


Typically generated by MACS2 or equivalent peak callers

Core Workflow (Step 1 + Step 2)
Step 1: Merge Hi-C Loop Outputs
library(ChIPSP)

hic_files <- list.files(
  pattern = "HiC\\.xls$",
  full.names = TRUE
)

hic_df <- mergeHiCLoops(hic_files)


Output

A merged data.frame containing all Hi-C loops across replicates and resolutions

Step 2: ChIP–Hi-C Spatial Integration and Ranking
chip_file <- list.files(
  pattern = "ChIP\\.xls$",
  full.names = TRUE
)

chipsp_results <- chipSPLink(
  chip_file = chip_file,
  hic_df    = hic_df
)


Output

A ranked data.frame containing:

chr, start, end

pileup (ChIP-seq signal)

FDR (Hi-C loop confidence)

Normalized scores

Final ChIP-SP ranking score

Optional: Export to BED Format
exportChipSPBed(
  chipsp_results,
  file = "ChIPSP_final_ranked_output.bed"
)


The BED output can be directly used for downstream annotation and visualization.

Downstream Analyses (Outside the Package)

Gene annotation, pathway analysis, and visualization are not part of the package API.

Reference scripts are provided in:

inst/scripts/


Including:

Step3_ChIP_SP_Gene_annotation_ChIPpeakAnno_UCSC.R

Step4_ChIP_SP_Gene_annotation_Pathwayanalysis_UCSC.R

Step5_ChIP_SP_Gene_annotation_KEGG_ChEA_UCSC.R

Typical downstream workflow:

Annotate BED output using ChIPpeakAnno

Perform pathway enrichment (KEGG, ChEA)

Submit gene lists to Enrichr

Visualize expression patterns using ClusterGVis and bulk RNA-seq data

Package Scope and Design Philosophy
Included

Core ChIP-SP spatial integration algorithm

Hi-C loop merging

Reproducible, non-destructive workflows

Excluded

Genome-specific annotation choices

Web-based enrichment tools

Visualization dependencies

This modular design ensures:

Reproducibility

Reviewer transparency

Compatibility with diverse downstream pipelines
